# 相关方状态看板优化需求说明书
## 1. 需求背景与目标
本次迭代主要针对“相关方状态看板”进行功能与体验优化。核心目标是：

- 提升数据可视化效果 ：更直观地展示作业状态与审批及时率。
- 优化用户交互体验 ：增强筛选、搜索、下钻查看详情的能力。
- 规范业务术语 ：同步更新最新的组织架构与部门名称。
## 2. 详细需求说明
### 2.1 状态分布展示优化
- 涉及模块 ：【当日总成/交车作业状态】
- 需求描述 ：将现有的状态展示优化为 状态分布情况 展示。
- 具体规则 ：必须明确区分并展示以下三种核心状态的分布（如饼图、柱状图或数字卡片）：
  1. 已进入
  2. 正在作业
  3. 作业完成
### 2.2 列表筛选与排序逻辑优化
- 涉及模块 ：【当日相关方作业明细】
- 需求描述 ：提升明细列表的查阅效率。
- 具体规则 ：
  1. 交互调整 ：筛选控件调整为“顶格固定”或“顶部置顶”，支持分类筛选查看。
  2. 默认排序调整 ：列表默认按照作业状态优先级排序，具体顺序为：
     - 第一优先级 ：正在作业
     - 第二优先级 ：已进入
     - 第三优先级 ：作业完成
     - 第四优先级 ：其他状态（置于最后）
### 2.3 全局时间维度搜索扩展
- 涉及模块 ：【全局界面搜索】
- 需求描述 ：增加时间维度的扩展搜索能力。
- 具体规则 ：看板界面需支持按不同的时间周期进行数据汇总和检索，包括： 每日 、 每周 、 每月 、 每季 、 每年 。
### 2.4 审批模块指标调整
- 涉及模块 ：【当日审批状态】及相关对比图表
- 需求描述 ：调整审批相关指标的展示维度，以突显“审批及时性”。
- 具体规则 ：
  1. 文案及维度修改 ：原【当日审批状态】修改为【当日审批部门】。
  2. 对比图表修改 ：原【当日审批状态与作业人数对比】修改为【申请数据与当日审批数据对比】。
  3. 业务目的 ：通过对比直观反映出当前审批是否及时（即：积压申请量与已审批量的差异）。
### 2.5 危险作业展示与下钻
- 涉及模块 ：【危险性作业】
- 需求描述 ：精简无效信息，增加数据穿透能力。
- 具体规则 ：
  1. 隐藏空数据 ：当危险作业数据为“无”（0）时，取消该项的显示。界面仅展示有实际危险作业的数据项。
  2. 详情下钻 ：支持列表或图表项的点击交互，用户点击具体的危险作业数据后，可进入（内点入）查看该危险作业的详细信息。
### 2.6 全局组织架构名词替换
- 涉及模块 ：【全局看板文案】
- 需求描述 ：根据最新公司组织架构，全局更新名词称呼。
- 具体规则 （包括但不限于以下替换）：
  - 事业部 统一更改为 制造中心
  - 总成车间 统一更改为 总成组装板块
  - 交车车间 统一更改为 调试交付板块
  - 其他要求 ：其他涉及的部门或板块名称，需开发/测试人员同步排查，并统一更新为现行称呼。
